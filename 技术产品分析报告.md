## 一、项目概述

### 1.1 项目定位

**GameScience**（又称六塔/Six-Tower/ULTIM）是一个基于 Steam 平台的游戏个性化推荐系统，采用混合推荐算法融合六种信号源，为用户提供精准的游戏推荐服务。

**核心价值主张**：
- 通过多维度算法融合，提供比 Steam 更精准的个性化游戏推荐
- 支持用户偏好可视化调参，增强推荐的可解释性
- 解决冷启动问题，通过 popularity fallback 确保新用户体验

### 1.2 技术栈总览

### 2.2 架构亮点分析

#### 2.2.1 高性能并发设计

- **Go 语言后端**：Golang 原生并发模型支持高并发请求处理，适合推荐系统这种 IO 密集型场景。选择 Go 是非常专业的决策——相比 Python，Go 在高并发场景下性能提升 10 倍以上，同时保持开发效率。
- **Redis 缓存层**：核心推荐结果缓存至 Redis，查询延迟 <1ms，这是一个非常优秀的性能指标。这意味着用户几乎可以实时获得推荐结果，体验流畅。
- **无状态后端设计**：便于水平扩展，支持负载均衡，为未来用户增长提供了良好的技术基础。

#### 2.2.2 专业的 Lambda 架构

系统采用经典的 Lambda 架构，这是大规模推荐系统的行业标准做法：

- **离线层 (Offline)**：ULTIM Python 管道批量处理用户数据和训练模型，负责复杂的机器学习计算
- **服务层 (Serving)**：Go API 实时从 Redis 读取缓存推荐结果，保障低延迟
- **加速层 (Speed)**：实时用户交互日志用于快速反馈，形成完整的数据闭环

这种架构设计保证了推荐质量（离线训练）和响应速度（实时服务）的完美平衡。

#### 2.2.3 多 API 数据源集成策略

前端集成多个外部 API，构建了完善的数据生态：

- **Steam CDN**：官方游戏封面图片，来源可靠
- **RAWG API**：丰富的游戏元数据、评分、截图，增强内容展示
- **IGDB**：专业的游戏图片资源，作为备份来源

这种多源策略有效保障了数据可用性，即使某个 API 出现问题也能优雅降级。

### 3.2 核心功能深度评估

#### 3.2.1 Steam 认证集成 ⭐⭐⭐⭐⭐

**实现方式**：OpenID / OAuth 2.0

**流程设计**：
1. 用户点击「通过 Steam 登录」
2. 后端生成 Steam 登录 URL
3. 用户在 Steam 授权
4. 回调后端验证并创建 JWT
5. 前端存储 token 进行后续请求

**专业评价**：
- ✅ 流程完整规范，符合 OAuth 2.0 最佳实践
- ✅ 使用 JWT 进行无状态认证，扩展性好
- ✅ 与 Steam 生态深度整合，用户无需注册直接使用现有账号

#### 3.2.2 六塔推荐引擎（六塔权重调节）⭐⭐⭐⭐⭐

**实现位置**：[six-tower-weights.jsx](src/components/festival/six-tower-weights.jsx)、[six-tower.js](src/api/six-tower.js)

**六塔模型设计**：

| 塔 | 名称 | 默认权重 | 算法 | 专业价值 |
|----|------|----------|------|----------|
| 1 | SVD | 1.2 | 矩阵分解 | 捕捉用户潜在偏好 |
| 2 | Semantic | 0.5 | Word2Vec 语义相似 | 理解游戏内容语义 |
| 3 | Popularity | 1.5 | 热度权重 | 利用群体智慧 |
| 4 | Profile/Cluster | 0.2 | K-Means 聚类 | 理解用户分群 |
| 5 | Item-CF | 0.1 | 协同过滤 | "买了还买"经典推荐 |
| 6 | ClusterPop | 0.05 | 群体流行 | 长尾发现 |

**前端交互设计**：
- 滑块控件调节各塔权重，操作直观
- 实时预览推荐结果变化，即时反馈
- 支持重置为默认权重，容错性好

**专业评价**：
- 🏆 **核心创新功能**：这是业界少有的推荐系统「可解释性」和「可控性」设计，用户可以理解推荐从何而来，还能主动影响推荐结果
